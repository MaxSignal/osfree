172/*
173Scan the string starting at ptr until the end of the next complete token,
174but do not scan past eptr. Return an integer giving the type of token.
175
176Return XML_TOK_NONE when ptr == eptr; nextTokPtr will not be set.
177
178Return XML_TOK_PARTIAL when the string does not contain a complete token;
179nextTokPtr will not be set.
180
181Return XML_TOK_INVALID when the string does not start a valid token; nextTokPtr
182will be set to point to the character which made the token invalid.
183
184Otherwise the string starts with a valid token; nextTokPtr will be set to point
185to the character following the end of that token.
186
187Each data character counts as a single token, but adjacent data characters
188may be returned together. Similarly for characters in the prolog outside
189literals, comments and processing instructions.
190*/
191
