38 @par Usage
39
40Every file and subdirectory has 4 attributes that can be turned on
41(set) or turned off (cleared): \a Archive, \a Hidden, \a Read-only,
42and \a System.
43
44The ATTRIB command lets you view, set, or clear attributes for any file,
45group of files, or subdirectory. You can view file attributes by
46entering ATTRIB without specifying new attributes (i.e.,
47without the \c [+|-[AHRS]] part of the format), or with the <tt>DIR
48/T</tt> command.
49
50You can view file attributes by entering ATTRIB without specifying new
51attributes (i.e., without the \c [+|-[AHRS]] part of the
52format). (You can also view file attributes with the :link reftype=hd
53refid=DIR.DIR:elink. /T command.)
54
55The primary use of ATTRIB is to set attributes. For example, you can set
56the read-only and hidden attributes for the file \c MEMO:
57:xmp.
58 [c&colon.\] attrib +rh memo
59:exmp.:p.
60Attribute options apply to the file(s) that follow the options on
61the ATTRIB command line. The example below shows how to set
62different attributes on different files with a single command. It
63sets the archive attribute for all :hp1..TXT:ehp1. files, then sets the
64system attribute and clears the archive attribute for :hp1.TEST.COM:ehp1. &colon.
65:xmp.
66 [c&colon.\] attrib +a *.txt +s -a test.com
67:exmp.:p.
68When you use ATTRIB on an HPFS drive, you must quote any file
69names which contain whitespace or special characters. See :link reftype=hd
70refid=FileNames.File Names:elink. for additional details.
71:p.
72To change directory attributes, use the :hp2./D:ehp2. switch. If you give
73ATTRIB a directory name instead of a file name, and omit :hp2./D:ehp2., it
74will append "\*.*" to the end of the name and act on all files in that
75directory, rather than acting on the directory itself.
76:p.
77Your operating system also supports "D" (subdirectory) and "V"
78(volume label) attributes. These attributes cannot be altered with
79ATTRIB; they are designed to be controlled only by the operating
80system itself.
81:p.
82ATTRIB will ignore underlines in the new attribute (the
83:hp2.[+|-[AHRS]]:ehp2. part of the command). For example, ATTRIB sees these
84two commands as identical&colon.
85:xmp.
86 [c&colon.\] attrib +a filename
87 [c&colon.\] attrib +__A_ filename
88:exmp.:p.
89This allows you to use a string of attributes from either the @ATTRIB
90variable function or from ATTRIB itself (both of which use underscores to
91represent attributes that are not set) and send that string back to ATTRIB
92to set attributes for other files. For example, to clear the attributes of
93:hp1.FILE2:ehp1. and then set its attributes to match those of
94:hp1.FILE1:ehp1. (enter this on one line)&colon.
95:xmp.
96 [c&colon.\] attrib -arhs file2 &amp. attrib +%@attrib[file1] file2
